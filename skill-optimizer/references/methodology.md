## 总览：优化的本质

Skill 的本质是**「被压缩、固化的专家决策经验」**——不是提示词，而是把专家做某件事的判断链、踩坑、阈值、分支固化下来，让 AI 可复现地执行。

「优化 Skill」= **知识工程 + 系统设计**，两大支柱：
- 支柱一：思维框架（想清楚·战略层）—— 6 个模型，决定方向是否正确
- 支柱二：工程方法库（做到位·战术层）—— 21 个手段，决定能否落地见效
- 贯穿：先量化指标，再闭环迭代

## 支柱一：6 个思维模型（按优先级）

### 1. 第一性原理 —— Skill 是「压缩的专家经验」
- 问「一个顶级专家做这件事时，脑子里的判断链是什么？」把**隐性判断显性化**。
- 反例：只写「分析竞品」，却没写判断标准、阈值、分支。

### 2. MECE 原则 —— 拆解不重不漏
- 价值链层面整体 MECE；单 Skill 内部：输入边界清晰、步骤互斥且穷尽、输出字段不重叠。

### 3. 输入-处理-输出（IPO）契约模型 —— 让 Skill 可级联
- 关键公式：`上游 OUTPUT 模板 == 下游 INPUT 模板`。
- 落地：契约标准化成固定 YAML/Schema，链才不断。

### 4. 决策树 / 状态机 —— 把「看情况」变成「if-then」
- 飘的根因是模糊指令。显式写 **触发条件 + 阈值 + 对应动作**。

### 5. 约束驱动设计 —— 用「护栏」保证下限
- 强制输出模板、强制数学推导链、强制合规校验、分层输出 P0/P1/P2 防截断。

### 6. 反馈闭环 / PDCA —— Skill 是迭代出来的
- Plan(设计) → Do(跑真实案例) → Check(对比专家基准) → Act(回填修正)。
- 每次踩坑写进 `SKILL.patch.md` → 复利沉淀。
