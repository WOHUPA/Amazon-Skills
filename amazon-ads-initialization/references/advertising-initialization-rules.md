# Amazon 新品广告初始化规则

## 1. 产品判定

### 数据增强判定优先级

| 数据来源 | 用途 | 触发条件 |
|---|---|---|
| SIF MCP `market_get_asin_keyword_signals` | 反查 ASIN 流量词、关键词贡献、自然排名稳定性、付费依赖 | 用户提供目标 ASIN 或竞品 ASIN |
| SIF MCP `ops_get_listing_traffic_overview` | 判断自然/广告流量占比、SP/SB/SBV 渠道结构 | 用户提供目标 ASIN 或竞品 ASIN |
| SIF MCP `ops_get_asin_sales_list` / `ops_get_asin_sales_trend` | 判断价格、销量趋势、生命周期和放量风险 | 用户提供目标 ASIN 或竞品 ASIN |
| SIF MCP `market_get_keyword_history` | 判断搜索量、ABA 排名、Top3 点击/转化集中度 | 用户提供关键词 |
| SIF MCP `market_get_keyword_demand` | 判断关键词生命周期、季节性、行动时机 | 用户提供关键词 |
| SIF MCP `market_get_keyword_root_trend` | 判断精确词覆盖率和长尾需求空间 | 用户提供关键词 |
| SIF MCP `market_get_keyword_competition` | 判断关键词竞争格局、可进入性、竞品 ASIN 流量份额 | 用户提供关键词 |
| SIF MCP `ads_get_asin_ad_structure` / `ads_get_asin_ad_traffic_trend` | 判断竞品广告结构、广告强度、SP/SB/SBV 分布 | 用户提供竞品 ASIN 或目标 ASIN |
| 用户输入 | 补充预算、库存、毛利线、关键词、竞品页数、主观市场判断 | SIF 数据缺失或需要人工边界 |

只调用 SIF MCP，不调用 SellerSprite 或其他第三方数据 MCP。不要编造 MCP 数据；没有取到数据时，明确标注 SIF 数据缺失，并回退到用户输入 + SOP 规则的纯规则模式。

CPC、bid、广告竞价、点击成本当前未在 SIF MCP 工具 schema 中明确暴露。涉及 CPC 判断时，必须使用用户输入、Amazon 广告后台建议竞价或纯 SOP 阈值，并标注数据来源。

### 蓝海 / 红海

| 类型 | 判定标准 | 初始化目标 |
|---|---|---|
| 蓝海 | 竞争小、销量低、竞品≤1页 | 拓流优先，尽快打开搜索词发现入口 |
| 红海 | 竞品多、销量高、竞品＞1页 | 精准卡位，控制无效流量并截流竞品 |

### 数据增强后的蓝海 / 红海判断

- 更偏蓝海的信号：
  - SIF 显示关键词搜索量有体量，但 Top3 点击/转化集中度不高。
  - SIF 显示关键词可进入性较高，前 20 ASIN 流量份额分散。
  - 竞品广告结构简单，主要依赖 SP，SB/SBV 覆盖弱或广告 Campaign 数量少。
  - 用户输入显示竞品≤1页、竞争小、销量低。
- 更偏红海的信号：
  - SIF 显示头部竞品广告强度高，SP/SB/SBV 多类型覆盖。
  - SIF 显示 Top3 点击/转化集中度高，关键词竞争格局偏封闭。
  - SIF 显示竞品流量或销量趋势稳定且广告依赖明显。
  - 竞品多、销量高、关键词竞争强，需要精准卡位和竞品截流。

### 生命周期

| 阶段 | 周期 | 评价 | 日销 | 广告销售占比 |
|---|---:|---:|---:|---:|
| 新品期 | 0-4周 | <10 | <5 | >60% |
| 推广期 | 5-12周 | 10-50 | 5-20 | 50% |
| 稳定期 | 12周+ | >50 | >20 | 5-30% |

## 2. 预算分配

### 生命周期预算基线

| 阶段 | 总预算 | SP 占比 |
|---|---:|---:|
| 新品期 | 销售额或目标销售额的 15-20% | 85-90% |
| 推广期 | 20-25% | 70-75% |
| 稳定期 | 10-15% | 60-65% |

### 蓝海产品

- 总预算：销售额的 10-15%；若处于新品期，可按新品期基线 15-20% 作为测试预算上限。
- SP 自动：60-70%，每日固定，不中断。
- SP 手动：30-40%，仅 3 个核心主词。
- ASIN 定向：5-10%，可选，不强制。
- 逻辑：蓝海竞争小，先拓流和积累数据，不应过早收窄。

### 红海产品

- 总预算：销售额的 20-30%；新品期必须先设人工预算上限。
- SP 自动：20-30%，仅拓词，低竞价。
- SP 手动：70-80%，覆盖大词和长尾词。
- ASIN 定向：15-20%，必开，用于截流竞品。
- 逻辑：红海流量昂贵，初始化要先控制相关性和卡位效率。

## 3. 广告结构搭建

### Campaign 单元化创建原则

所有初始化广告结构必须优先保证数据归因清晰，按以下硬规则拆分：

| 广告类型 | 创建原则 | 说明 |
|---|---|---|
| SP 自动 | 1 个 Campaign = 1 个广告组，四种自动定向可同时开启 | 自动广告允许在同一个广告组内同时开启紧密匹配、宽泛匹配、同类商品、关联商品 |
| SP 手动关键词 | 1 个 Campaign = 1 个广告组 = 1 个关键词 | 不允许一个广告组放多个关键词 |
| SP 手动同词多匹配 | 1 个 Campaign = 1 个广告组 = 1 个关键词 + 1 种匹配方式 | 同一关键词的精准、词组、广泛必须拆开 |
| ASIN 定向 | 1 个 Campaign = 1 个广告组 = 1 个 ASIN 定向目标 | 多个竞品 ASIN 必须逐个拆分 |

输出 Campaign 初始化结构时，必须列出广告类型、Campaign 名称建议、广告组名称建议、关键词或定向目标、匹配方式、预算占比和用途；SP 自动展示四种自动定向开启状态。
新品广告初始化不投放 SD 广告，不输出 SD 预算、SD 定向或 SD Campaign 结构。

### 蓝海结构

- SP 自动：
  - 4 种匹配全开。
  - 仅否定完全无关词。
  - 作用：发现搜索词、积累点击和转化样本。
- SP 手动：
  - 选择 3 个核心词。
  - 每个核心词单独创建 Campaign 和广告组。
  - 默认每个核心词拆成精准 Campaign 和词组 Campaign。
  - 搜索量 <6000 时，可为该词额外拆一个广泛匹配 Campaign。
- ASIN 定向：
  - 选择 5-10 个同类竞品。
  - 每个竞品 ASIN 单独创建 Campaign 和广告组。
  - 可选开启，不作为强制初始化动作。

### 红海结构

- SP 自动：
  - 仅开启紧密匹配。
  - 竞价 = 建议价的 50-70%。
  - 泛大词列入初始否词候选，需人工确认后再添加。
  - 初始观察窗口：2-3 天，仅作为上线前计划，不输出优化动作。
- SP 手动：
  - 大词和长尾词都按单关键词单 Campaign 拆分。
  - 大词：每个大词单独创建精准匹配 Campaign，可设置首页溢价 50-100%。
  - 长尾词：每个长尾词分别创建精准 Campaign 和词组 Campaign。
- ASIN 定向：
  - 选择 TOP10 竞品。
  - 每个竞品 ASIN 单独创建 Campaign 和广告组。
  - 竞价 = 建议价的 70-80%。
  - 必须开启。

## 4. 初始化边界

本 Skill 只输出广告初始化方案，不输出上线后的广告优化动作。

允许输出：

- 初始预算比例和每日预算拆分。
- 初始 Campaign / Ad Group 结构。
- 初始匹配方式、初始竞价口径和首页溢价口径。
- 初始关键词、长尾词、竞品 ASIN、品牌词边界和初始否词候选。
- 上线前人工确认项。

禁止输出：

- 上线后的否词动作，例如“点击 >20 且无转化立即否定”。
- 上线后的调价动作，例如“降低竞价 20-30%”或“提高优质词竞价”。
- 上线后的 BD+ 溢价优化、广告位报告复盘、ACOS/CVR/CTR 优化。
- 每日、每周、每月复盘节奏。
- 基于投放后表现的优质词、潜力词、低效词、无效词分级。

## 5. 初始化关键词分层

| 词类型 | 初始化来源 | 初始用途 |
|---|---|---|
| 核心词 | 用户提供主词、SIF 反查主流量词、关键词需求强且相关度高的词 | SP 手动精准 / 词组启动 |
| 长尾词 | SIF 词根覆盖率低、搜索表达分散、用户提供场景词 | SP 手动词组 / 精准补充 |
| 竞品词 | SIF 关键词竞争工具返回的头部 ASIN 或用户提供竞品 ASIN | ASIN 定向候选 |
| 品牌词 | 明确品牌相关或 SIF 判断为品牌/精准词 | 默认不主攻，除非用户确认有授权或截流策略 |
| 初始否词候选 | 明显不相关词、错误品类词、禁投词 | 仅作为上线前人工确认，不自动否定 |
